PK 02 Normalización

Normalización de datos

Objetivo

Ver cómo una tabla de empleados con datos mezclados, repetidos y multivaluados puede transformarse en varias tablas relacionadas mediante claves primarias y claves foráneas.

Idea clave

Normalizar no es partir una tabla porque sí: es separar datos según su significado para reducir repetición, evitar inconsistencias y permitir relaciones claras entre entidades.

Tabla sin normalizar

La hoja original guarda todos los datos del empleado en una sola estructura. Eso la vuelve fácil de leer al principio, pero difícil de mantener cuando aparecen datos repetidos o más de un valor para el mismo campo.

Proceso paso a paso

La normalización se puede pensar como una serie de preguntas simples. En cada paso miramos qué hay en la tabla original, qué problema trae y cómo queda en el modelo normalizado.

Regla práctica

Si un dato se repite, se mezcla con otro dato o puede aparecer varias veces para la misma persona, probablemente conviene separarlo.

Paso activo

Dar un identificador a cada empleado

Antes

La fila no tiene una clave propia. Para reconocer a un empleado dependemos de datos como DNI o nombre.

Después

Se agrega IDEmpleado como clave primaria de la tabla Empleados.

Por qué mejora

Con una clave primaria podemos relacionar teléfonos, emails, ciudad y área con la persona correcta sin copiar todos sus datos.
