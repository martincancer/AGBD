PK 01 Introducción

Bases de datos: conceptos iniciales

Objetivo

Comprender qué es una base de datos, por qué se utiliza un sistema de gestión de bases de datos y cuáles son las diferencias principales entre bases relacionales y no relacionales.

Idea clave

Una base de datos permite almacenar información de forma ordenada para recuperarla, analizarla, transmitirla y mantenerla con mayor consistencia que en una lista aislada.

Qué es una base de datos

Una base de datos es un conjunto de información perteneciente a un mismo contexto, organizada de manera sistemática y almacenada electrónicamente. Puede contener distintos tipos de datos: texto, números, imágenes, archivos u otros registros necesarios para una aplicación.

Muchas bases comienzan como una lista en una hoja de cálculo o en un procesador de texto. Cuando esa lista crece, suelen aparecer redundancias, inconsistencias y dificultades para buscar o extraer subconjuntos de información.

Cuando los datos crecen Qué problema aparece Qué aporta una base de datos
Hay muchas filas repetidas o parecidas Redundancia de datos Diseño organizado y menor duplicación
La información se actualiza en varios lugares Inconsistencias Reglas de integridad y actualización controlada
Se necesitan búsquedas más precisas Extracción limitada de datos Consultas, filtros y relaciones entre tablas

DBMS o sistema de gestión

Un DBMS es un sistema de administración de bases de datos. Su función es facilitar el almacenamiento ordenado, la recuperación rápida, la protección y la manipulación de la información.

Cuando una lista empieza a ser difícil de mantener, pasarla a una base de datos creada con un DBMS ayuda a organizar los datos, controlar accesos y evitar errores frecuentes.

Tipos principales de bases de datos

Existen varios modelos, pero para esta materia vamos a distinguir dos familias importantes: bases relacionales o SQL, y bases no relacionales o NoSQL.

Tipo Cómo organiza los datos Uso habitual
SQL o relacional Usa tablas con filas y columnas que pueden relacionarse mediante claves. Aplicaciones donde importa la estructura, la consistencia y las consultas con SQL.
NoSQL o no relacional No sigue necesariamente el esquema tradicional de tablas; puede usar documentos, clave-valor, grafos u otros modelos. Grandes volúmenes de datos distribuidos o estructuras más flexibles y cambiantes.
Criterio de elección

No hay un tipo universalmente mejor. La elección depende de las necesidades del proyecto, la forma de los datos, el volumen esperado y las operaciones que se van a realizar.

Bases de datos relacionales

Una base de datos relacional organiza la información en tablas relacionadas entre sí mediante claves primarias y claves externas.

  • Cada tabla representa un conjunto de datos sobre una entidad o concepto.
  • Cada fila representa un registro único dentro de la tabla.
  • Cada columna representa un atributo de esa entidad.
  • La clave primaria identifica de manera única cada fila.
  • La clave externa permite vincular una tabla con otra.

Ventajas de las bases relacionales

Ventaja Qué significa
Estructura organizada La forma tabular permite guardar los datos de manera ordenada y consultable.
Integridad de datos Las relaciones entre tablas ayudan a mantener coherencia y evitar duplicación innecesaria.
Flexibilidad de consulta SQL permite realizar búsquedas, filtros y combinaciones entre múltiples tablas.
Escalabilidad Pueden crecer para manejar grandes volúmenes de datos y usuarios concurrentes.
Seguridad Los DBMS ofrecen mecanismos de autenticación, autorización y control de acceso.
Compatibilidad con estándares SQL es un lenguaje ampliamente aceptado y facilita la interoperabilidad con otras herramientas.

Desventajas a considerar

  • Complejidad de diseño: los sistemas con muchas tablas y relaciones requieren una planificación cuidadosa.
  • Rendimiento en escalas extremadamente grandes: las consultas complejas y la integridad referencial pueden impactar en bases muy grandes.
  • Rigidez en la estructura: los cambios frecuentes en la forma de los datos pueden ser más difíciles que en algunos modelos NoSQL.
  • Escalabilidad horizontal: sumar servidores puede ser más complejo que aumentar recursos de hardware en un único servidor.
Para recordar

Una base relacional no empieza en SQL: empieza en el diseño. Antes de escribir consultas conviene entender qué entidades existen, qué atributos tienen y cómo se relacionan.
